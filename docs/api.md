# API Reference

Interactive docs (generated by drf-spectacular) are served live at
**`/api/docs/`** (Swagger UI) with the raw schema at **`/api/schema/`**. This
file is the human summary.

Base URL (local): `http://localhost:47080`

## Authentication

- **API clients** use JWT (SimpleJWT). Obtain a token, then send
  `Authorization: Bearer <access>`.
- **Browser UI** uses Django sessions (login at `/accounts/login/`).

| Method | Path | Auth | Description |
|---|---|---|---|
| POST | `/api/auth/register` | none | Create an account **and onboarding profile**. Body: `email`, `password`, `full_name`, `age`, `place_of_living`, `gender`, `country_of_origin`, `description?`. All profile fields except `description` are required. |
| POST | `/api/auth/login` | none | Obtain JWT. Body: `email`, `password` â†’ `{access, refresh}`. |
| POST | `/api/auth/refresh` | none | Exchange a refresh token for a new access token. |
| GET | `/api/auth/me` | JWT | Current user + profile. |
| PATCH/PUT | `/api/auth/me` | JWT | Update the onboarding profile (`age`, `place_of_living`, `gender`, `country_of_origin`, `description`, `full_name`). Future submissions snapshot the latest values. |

Auth endpoints are rate-limited (`auth` scope, 10/min) â€” safety rule #4.

## Submissions (authenticated user)

| Method | Path | Description |
|---|---|---|
| POST | `/api/submissions` | Create a submission â€” **photo only** (`multipart/form-data`: `photo`). Metadata (name/age/place/gender/country/description) is **snapshotted from the user's profile** at creation. Returns `201` (status `pending`), or `400` if the profile is incomplete. Upload-throttled. |
| GET | `/api/submissions` | List **your** submissions (paginated). |
| GET | `/api/submissions/{id}` | Retrieve one (owner or admin only). Includes a presigned `photo_url`. |
| GET | `/api/submissions/{id}/result` | Lifecycle + result only: `{status, classification_result, error_detail}`. Polled by the UI. |

`gender` âˆˆ `male | female | other | undisclosed`. `age` âˆˆ `0..120`.

### Example

```bash
BASE=http://localhost:47080
curl -X POST $BASE/api/auth/register -H 'Content-Type: application/json' \
  -d '{"email":"u@example.com","password":"Str0ngPass!9","full_name":"Alice",
       "age":29,"place_of_living":"Berlin","gender":"female","country_of_origin":"Germany"}'

TOKEN=$(curl -s -X POST $BASE/api/auth/login -H 'Content-Type: application/json' \
  -d '{"email":"u@example.com","password":"Str0ngPass!9"}' | jq -r .access)

# Photo-only â€” metadata is taken from the profile created above.
curl -X POST $BASE/api/submissions -H "Authorization: Bearer $TOKEN" -F photo=@photo.jpg
```

## Admin (staff only â€” `403` otherwise)

| Method | Path | Description |
|---|---|---|
| GET | `/api/admin/submissions` | List/filter/search **all** submissions (paginated). |
| GET | `/api/admin/submissions/{id}` | Full record: metadata, presigned photo, result, timestamps. |

Filter query params: `age_min`, `age_max`, `gender`, `location` (matches
place of living), `country`, `status`, `q` (free-text over name + description),
`page`.

```bash
curl "$BASE/api/admin/submissions?gender=female&age_min=18&age_max=40&country=Germany" \
  -H "Authorization: Bearer $ADMIN_TOKEN"
```
